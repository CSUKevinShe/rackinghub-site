# RackingHub Planner — 2D 三视图质量对标

## 关键发现：标杆的三视图特点

看标杆 warehouse-planner.com 的实际工具界面，关键特点：

### 1. 多视图布局
- **上面**：大尺寸俯视图（Plan View），占据主要空间
- **下面左侧**：正立面图（Front Elevation），展示层高和横梁
- **下面右侧**：侧视图（Side Elevation）
- 三个视图**同时显示**，不是分开的卡片

### 2. 视觉风格
- **货架颜色层次**：
  - 货架货位：**tan/beige 色块**
  - 立柱（Uprights）：**蓝色竖线**
  - 横梁（Beams）：**黄色横线**（立面图中清晰可见）
  - 建筑柱子/墙：**绿色粗条**
- **背景网格**：浅灰色网格，辅助对齐
- **标注线**：清晰的尺寸标注，数字标注在尺寸线旁边
- **叉车图标**：红色小叉车图标，增加尺度感

### 3. 控件风格
- 数字输入框（带 +/- 微调按钮）
- 不是滑块，是精确的数字输入
- 布局紧凑，分组清晰

### 4. 信息呈现
- 俯视图中标注了具体的尺寸数字（1500, 3000, 2700, 11200）
- 立面图中清晰显示层高（6500, 7500）
- 叉车图标在右下角

## 我们的问题

对比标杆，我们的三视图差距：
1. **货架表示单一** — 只有蓝色块，没有 tan 色货位、黄色横梁、绿色柱子的层次
2. **标注不够清晰** — 尺寸线样式和标杆不同，数字可读性差
3. **视图布局** — 标杆的三个视图是紧凑排列的，我们的是分开的卡片
4. **缺少叉车图标** — 标杆有红色叉车作为尺度参考
5. **背景网格** — 标杆有清晰的网格辅助线

## 任务

按标杆的视觉风格重写 Canvas 渲染（layout-engine.js 中的 drawTopView/drawFrontView/dSideView）：

1. **货架颜色层次**：
   - 货位/托盘：tan/beige (#d4a574 或类似)
   - 立柱：蓝色竖线 (#1a365d)
   - 横梁：黄色横线 (#f59e0b 或类似)
   - 建筑柱子：绿色粗条 (#10b981 或类似)

2. **背景网格**：加浅灰色网格线

3. **标注优化**：尺寸线清晰，数字标注可读

4. **叉车图标**：在俯视图或侧视图中加一个小叉车图标（可以用 Canvas 绘制简单的）

5. **视图布局**：三个视图紧凑排列，标杆的风格

**不要改业务逻辑，只改 Canvas 渲染样式。**

完成后本地 http://localhost:8080/planner/ 验证截图。
